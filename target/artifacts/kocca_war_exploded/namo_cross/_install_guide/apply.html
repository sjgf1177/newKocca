CrossEditor

크로스에디터를 사용하여 폼(form) 전송 구현하기

  • 크로스에디터는 다양한 서버사이드 언어를 통해 다양하게 접목시킬 수 있습니다. 여기서는 크로스에디터에서 작성한 콘텐츠를 폼(form) 전송하는 방법에 대해 알아보겠습니다.
  • 순서
    1) hidden 객체 생성 하기
    2) 크로스에디터 콘텐츠 작성 유무 확인 하기
    3) 크로스 에디터에서 작성한 콘텐츠의 내용 가져오기
    4) 작성된 내용을 크로스에디터에 삽입하기

  • 1. hidden 객체 생성 하기

    크로스에디터에서 콘텐츠를 작성하는 것과는 별도로 폼(from)을 통해 전송할 hidden 객체가 필요합니다. hidden 객체로는 주로 "<input type=”hidden” …..>"과 "<textarea id=""…..>" 를 사용할 수 있는데, 크로스에디터에서는 작성된 콘텐츠의 내용을 소실없이 전송할 수 있는 "textarea"의 사용을 권장합니다.
  • Ex)
    <textarea id="editorValue" name="editorValue" style="display: none"></textarea>

  • 2. 크로스에디터 콘텐츠 작성 유무 확인 하기

    폼(form)전송을 할 경우 콘텐츠의 내용의 유무에 대한 구분이 필요한 경우가 있습니다.
    이러한 경우 크로스에디터에서 제공하는 "IsDirty" 또는 "GetTextValue" Method를 사용하여 구별을 할 수 있습니다.

    "IsDirty" Method 는 문서 내용이 변경 되었을 경우 "true" 값을, 문서 변경이 없을 경우 "false"의 값을 반환합니다.
    "GetTextValue" Method 는 문서 안의 text 값을 반환하기 때문에 "IsDirty"와 같은 기능을 할 수 있습니다.
    크로스에디터로 focou이동은 "SetFocusEditor" Method를 사용하면 됩니다.
  • Ex) 1) IsDirty
    if(!CrossEditor.IsDirty()){ // 크로스에디터 안의 콘텐츠 입력 확인
          alert(" 에디터에 내용을 입력해 주세요 !!");
          CrossEditor.SetFocusEditor();// 크로스에디터 Focus 이동
          return false;
          }
  • 2) GetTextValue
    if(CrossEditor.GetTextValue() == ""){ // 크로스에디터 안의 콘텐츠 입력 확인
          alert(" 에디터에 내용을 입력해 주세요 !!");
          CrossEditor.SetFocusEditor();// 크로스에디터 Focus 이동
          return false;
          }

  • 3. 크로스 에디터에서 작성한 콘텐츠의 내용을 가져오기

    크로스에디터에서 작성된 콘텐츠의 내용을 가져오기 위해서는 크로스에디터에서 제공하는 Method 중 “GetValue” 나 “GetBodyValue” 를 사용하면 됩니다. 반환된 내용은 HTML 양식으로 반환합니다.
    Method에 대한 자세한 내용은 개발자 도움말의 API 안내를 참고하십시오.

    GetValue: 크로스에디터에서 작성된 콘텐츠의 내용을 모두 반환합니다.
    GetBodyValue: 크로스에디터에서 작성된 콘텐츠 내용 중 Body 안의 내용을 반환합니다.
  • Ex) 1) GetValue
    document.getElementById("editorValue").value = CrossEditor.GetValue();
  •       2) GetBodyValue
    document.getElementById("editorValue").value = CrossEditor.GetBodyValue();

  • 4. 작성된 내용을 크로스에디터에 삽입하기

    콘텐츠 내용을 크로스에디터에 삽입할 경우에는 크로스에디터에서 제공하는 Method 중 SetValue나 SetBodyValue를 사용하면 됩니다.
    단 크로스에디터에서 내용을 가져오는 Method와 삽입 시 사용하는 Method는 서로 짝을 이루고 있습니다. GetValue로 내용을 가져왔을 경우에는 SetValue를 사용해야 하며 GetBodyValue를 사용하여 가져왔을 경우에는 SetBodyValue를 사용하여야 합니다.

    GetValue -> SetValue
    GetBodyValue -> SetBodyValue
  • 1) 작성된 콘텐츠의 내용을 JavaScript를 사용하여 에디터에 삽입 시 콘텐츠의 내용 중 줄 구분자나 더블 쿼테이션 (" ") , 싱글 쿼테이션 ('')으로 인해 JavaScript 구문이 오류가 나 정상 작동을 하지 않는 경우가 있습니다. 이러한 경우를 미연에 방지 하기 위해 Hidden 객체를 생성하여 콘텐츠의 내용을 삽입 후 자바스크립트에서 그 값을 참조하면 됩니다.
  • Ex)
    <textarea id="contents" name="contents" style="display: none"> 콘텐츠 내용 </textarea>
  • 2) 생성한 Hidden 객체를 참조하여 크로스에디터에 SetValue 또는 SetBodyValue를 사용하여 콘텐츠를 삽입합니다.
    에디터 초기화 후 바로 Method를 호출 시 환경에 따라 정상적으로 호출이 되지 않는 경우가 있습니다. 이러한 경우를 미연에 방지 하기 위해 크로스에디터에서는 에디터 초기화 후 발생되는 "OnInitComplate" 이벤트를 사용하여 Method를 실행 시키기를 권장 합니다.
  • Ex) (1) SetValue
    var contentValue = document.getElementById("contents").value; // Hidden 값 참조
    var CrossEditor = new NamoSE('namoeditor1');
    CrossEditor.editorStart();

    function OnInitComplate(){
      CrossEditor.SetValue(contentValue); // 콘텐츠 내용 에디터 삽입
    }
  •       (2) SetBodyValue
    var contentValue = document.getElementById("contents").value; // Hidden 값 참조
    var CrossEditor = new NamoSE('namoeditor1');
    CrossEditor.editorStart();

    function OnInitComplate(){
      CrossEditor.SetBodyValue(contentValue); // 콘텐츠 내용 에디터 삽입
    }

에디터 최초 실행 시 바로 출력하지 않는 경우(display None)

  • 크로스에디터가 최초 실행 시 비즈니스 로직으로 인해 화면에 출력되지 않는 경우(display None)가 있을 것입니다. 이러한 경우 크로스에디터의 높이가 계산되지 않아 크로스에디터를 화면에 출력(display)했을 때 정상적으로 출력이 되지 않습니다.
    여기에서는 크로스에디터를 최초 실행 시 화면에 출력되지 않는 경우(display None) 적용 방법에 대해 알아 보겠습니다.
  • 순서
    1) 크로스에디터 포커스 설정
    2) 크로스에디터 displayNone target 설정

  • 1. 크로스에디터 포커스 설정

    페이지 호출 시 크로스에디터가 화면에 출력되지 않는 경우(display None) 크로스에디터에 focus 설정이 필요 없기 때문에 "SetFocus"을 사용하여 focus 설정을 없애 줍니다.
    크로스에디터에서 focus 설정은 "params"의 "SetFocus"로 가능 합니다.
    SetFocus : true -> 크로스에디터 focus 설정
                    false -> 크로스에디터 focus 취소
  • Ex)
    var CrossEditor = new NamoSE('namoeditor1');
    CrossEditor.params.SetFocus = false; // 에디터 포커스 설정
    CrossEditor.editorStart();

  • 2. 크로스에디터 DisplayNoneTarget 설정

    크로스에디터가 최초 실행 시 화면에 출력되지 않는 경우(display None) 에디터의 출력 높이를 계산하기 위해 display None이 적용될 Element(예를들어 크로스에디터를 감싸고 있는 DIV 레이어)를 “params”에서 제공하는 “DisplayNoneTarget”을 사용하여 지정 해주어야 합니다.
  • Ex)
    <div id="crossEditor" style="display: none">
          <script language="javascript" >
               var CrossEditor = new NamoSE('namoeditor1');
               CrossEditor.params.SetFocus = false; // 에디터 포커스 설정
               CrossEditor.params.DisplayNoneTarget = document.getElementById("crossEditor");
               // 크로스에디터를 감싸고 있는 레이어 설정
               CrossEditor.editorStart();
          </script>
    </div>

참고

    크로스에디터 배포판을 서버에 설치 후 해당 도메인을 통해 "index.html"을 접속하시면 크로스에디터의 구동 사항을 확인할 수 있습니다.
