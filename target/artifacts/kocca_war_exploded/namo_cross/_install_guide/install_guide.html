CrossEditor

설치사양

지원 운영체제
  • Microsoft Windows 2000 Server or Higher
  • Linux
  • Unix
지원 HTML
  • HTML 2.0 or Higher
  • XHTML 1.0 or Higher
지원 웹 언어
  • ASP
  • ASP.NET 2.0 or Higher
  • PHP 4.3.0 or Higher
  • JSP JDK 1.4.2 or Higher
지원 웹 서버
  • IIS 5.0 or Higher
  • Microsoft .NET Framework Version 2.0 or Higher
  • Apache 2.0 or Higher
  • tomcat 4.1 or Higher

언어별 설치 주의 사항

1) php
php.ini 환경설정 파일의 "allow_url_fopen = On" 설정확인 후 설정이 "Off" 인 경우 "On" 으로 변경시킵니다
2) jsp
cos.jar 파일을 설치합니다. (다운로드 사이트 : http://servlets.com/cos/)
<설치 방법 : tomcat 4.1을 기준>
1. 파일 다운 로드 후 압축 해제 합니다.
2. lib 폴더 안에 cos.jar 파일 복사 합니다.
3. 해당 어플리케이션 안의 WEB-INF/lib 또는 톰캣 설치 폴더/common/lib 안에 복사 합니다.



3) ASP.NET
배포된 설치파일중 "web.config"파일이 구성설정 파일로 등록되어야만 에디터 실행이 정상적으로 이루어집니다.

파일 초기 설정

1) 쓰기 권한 설정
크로스에디터는 설정 된 내용을 파일 쓰기를 통하여 저장을 하고 있습니다. 아래의 경로에 위치한 파일에 쓰기 권한을 추가 합니다.
파일명
  • 크로스에디터 설치폴더 경로/manage/asp/ manageInfo.asp
  • 크로스에디터 설치폴더 경로/manage/jsp/ manageInfo.jsp
  • 크로스에디터 설치폴더 경로/manage/php/ manageInfo.php
  • 크로스에디터 설치폴더 경로/manage/asp.net/ manageInfo.aspx
  • 크로스에디터 설치폴더 경로/config/xmls /config.xml

일반 설치 방법

  • 1) 설치를 원하는 파일의 상단에 해당 파일의 HTML에서 문서 형식(Doctype) 지정 합니다.
  • 2) "<head></head>" 태그 사이에 아래의 코드를 추가 합니다
    <script type="text/javascript" src=""></script>
    코드 추가 후 src 값에"namo_scripteditor.js" 의 경로를 입력 합니다.
    "<script type="text/javascript"
    src="크로스에디터 설치폴더 경로/js/namo_scripteditor.js"></script>"
  • 3) "<body>" 태그 안쪽의 에디터가 들어갈 위치에 에디터 생성 코드를 삽입 합니다.
    <script type="text/javascript">
    var CrossEditor = new NamoSE("임이의 이름");
    CrossEditor.editorStart();     </script>
    크로스에디터 생성 변수의 이름은 변경이 가능합니다.
  • 4) 크로스에디터 초기 설정을 위해 관리자 페이지에 접속을 합니다.
    관리자 페이지는 경로는 "구동 서버 URL/크로스에디터 설치 폴더 경로/manage" 입니다. 아래의 화면이 출력되면 정상적인 접속 입니다.

  • 5) 관리자 페이지 접속 후 로그인을 합니다. 아이디와 패스워드는 배포 시 지급된 내용을 참고 합니다.
    상단의 메뉴를 통해 관리자 설정 페이지로 이동을 합니다 환경 설정 부분과 화면 설정 부분을 설치 시스템에 맞게 입력 후 하단의 저장 버튼을 선택합니다.

  • 6) 저장 완료 메시지 확인 후 상단의 메뉴를 통해 미리보기 페이지로 이동을 합니다.
    미리보기 화면에서 크로스에디터의 정상적인 구동과 환경 설정 및 화면 설정 사항을 확인 합니다.
  • 7) 위의 모든 사항이 완료되면 크로스에디터의 설치가 완료 됩니다.

Servlet 설치 방법

  • 크로스에디터 servlet 버전에서 에디터가 정상적으로 동작을 하기 위해서는 에디터가 설치된 물리적 경로와 서블릿 컨텍스트 경로를 입력해 주어야 합니다. 크로스에디터 설치가 필요한 서버에 업로드후 아래의 순서에 따라 크로스에디터를 설치해 주시면 됩니다.
  • 순서
    1) 크로스에디터 파일 업로드
    2) 관리자 페이지 접근
    3) 서블릿 Context 경로 및 크로스에디터 web server 설치 경로 입력
    4) 원하는 페이지에 크로스에디터 적용

  • 1. 크로스에디터 파일 업로드

    크로스에디터 설치가 필요한 서버에 크로스에디터 파일을 업로드후 "크로스에디터 설치 경로/websource/servlet" 폴더로 접근 합니다.
    1) servlet 하위 폴더안에 있는 classes 폴더를 "$WebApplication_Root/WEB-INF/" 안에 복사 합니다.
    2) servlet 하위 폴더안에 있는 lib 폴더를 "$WebApplication_Root/WEB-INF/" 안에 복사 합니다.
    3) "$WebApplication_Root/WEB-INF/" 안의 "web.xml" 에 servlet mapping을 추가 합니다.
  • <tomcat 기준 설정>
    <servlet>
          <servlet-name>CrosseditorServlet</servlet-name>
          <display-name>CrosseditorServlet</display-name>
          <description></description>
          <servlet-class>
               com.namo.crosseditor.controller.CrosseditorServlet
          </servlet-class>
    </servlet>
    <servlet-mapping>
          <servlet-name>CrosseditorServlet</servlet-name>
          <url-pattern>/CrosseditorServlet</url-pattern>
    </servlet-mapping>

  • 2. 관리자 페이지 접근

  • 크로스에디터 초기 설정을 위해 관리자 페이지에 접근 합니다. 관리자 페이지의 접속 경로는 “구동서버 URL/크로스에디터 설치 폴더 경로/manage” 입니다. 아래와 같은 화면이 출력되면 정상적인 접속입니다.

  • 3. 서블릿 Context 경로 및 크로스에디터 설치 경로 입력

    먼저 컨텍스트 경로 입력란에 서블릿을 접속할 수 있는 컨텍스트 경로를 URL 형식으로 입력 합니다.
    예를 들어 서블릿을 접속 할 수 있는 URL 경로가 "http://www.test.co.kr/crosseditor" 인 경우 "/crosseditor" 를 입력해 주시면 됩니다. 설정된 컨텍스트 경로가 없을 경우 "/" 값만 입력합니다.
    컨텍스트 경로 입력이 완료 되면 설치폴더 경로를 입력하시면 됩니다. 설치 폴더 경로는 크로스에디터가 설치된 물리적 경로를 입력해 주시면 됩니다. 만일 web서버와 was 서버가 분리가 되어 있는 경우 was 서버에서 웹서버의 크로스에디터 설치 폴더에 접근 할 수 있는 Mount 경로를 입력합니다.

  • 입력 후 확인을 선택 합니다.
    배포 시 지급된 관리자 아이디와 비밀번호로 로그인한 후 관리자 설정 화면이 출력 되면 크로스에디터의 설치 경로 설정이 완료 된 것입니다.
  • * 이미지 경로 설정
    기본적으로 제공하는 이미지 저장 경로가 아닌 다른 이미지 저장 경로를 설정할 때 이미지가 저장될 server의 물리적 경로와 물리적 경로에 접근 할 수 있는 URL 경로를 적어 주어야 합니다. 두 가지 정보를 모두 입력해야 정상적으로 입력이 됩니다. 이미지 저장 URL 경로는 Full URL 경로나 절대 경로를 입력 하시면 됩니다.

  • 4. 크로스에디터 적용

  • 1) 설치를 원하는 파일의 상단에 해당 파일의 HTML에서 문서 형식(Doctype)을 지정합니다.
  • 2) "<head></head>" 태그 사이에 아래의 코드를 추가 합니다
    "<script type="text/javascript"
    src=" 크로스에디터 설치폴더 경로 /js/namo_scripteditor.js"></script>"
  • 3) "<body>" 태그 안쪽의 에디터가 들어갈 위치에 에디터 생성 코드를 삽입 합니다.

    코드 삽입 시 "<textarea>" 태그도 함께 사용하시기를 권장합니다.
    브라우저의 자바스크립트 기능이 비활성화 되어 있는 경우 또는 자바스크립트가 정상적으로 동작 하지 않는 환경에서 크로스에디터가 실행되지 않아 글 작성을 할 수 없습니다. 이러한 경우에도 정상적으로 글 작성을 할 수 있도록 "<textarea>"를 함께 사용하는 것이 웹 접근성을 높여 주는 방법이며 W3C의 권고 사항 입니다.

    위와 같이 사용하는 방법은 크로스에디터 실행시 호출하는 클래스에 지정하는 이름과 "<textarea>"의 "id" 속성값을 동일하게 지정하면, 자바스크립트가 정상적으로 동작하는 환경에서는 크로스에디터 실행과 동시에 "<textarea>" 엘리먼트는 "Display None"이 되어 화면에 출력되지 않습니다.

    이는 권장 사항 일뿐 "<textarea>"와 함께 사용하지 않아도 정상적인 브라우저 환경에서는 아무런 문제가 되지 않습니다.
    <textarea id="namoeditor1" name="namoeditor1"></textarea>
    <script type="text/javascript">
    var CrossEditor = new NamoSE("namoeditor1");
    CrossEditor.editorStart();
    </script>
